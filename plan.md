## ФАЗА 6 — Мультиязычность фронтенда

**Цель:** URL-структура `/ru/`, `/en/`, `/hy/`, переключатель языков в Header, все тексты UI переведены.

### 6.1 Установка и настройка next-intl
```bash
npm install next-intl
```

**Файлы для создания:**
- `src/i18n/routing.ts` — конфигурация локалей (`ru`, `en`, `hy`)
- `src/middleware.ts` — перехват запросов и редирект на язык
- `src/i18n/request.ts` — определение текущей локали
- `messages/ru.json` — статические тексты UI на русском
- `messages/en.json` — на английском
- `messages/hy.json` — на армянском

**Пример `messages/ru.json`:**
```json
{
  "nav": { "home": "Главная", "services": "Услуги", "team": "Команда", "blog": "Блог", "cases": "Дела", "about": "О нас", "contact": "Контакты" },
  "hero": { "call": "Позвонить" },
  "sections": { "readMore": "Читать далее", "viewAll": "Смотреть все", "allArticles": "Все статьи" },
  "contact": { "name": "Имя", "email": "Эл. почта", "phone": "Телефон", "message": "Сообщение", "send": "Отправить" },
  "common": { "notFound": "Страница не найдена", "back": "Назад" }
}
```

### 6.2 Структура маршрутов
```
src/app/
├── [locale]/               # NEW — обёртка для всех публичных страниц
│   ├── layout.tsx          # Передаёт locale в next-intl provider
│   ├── page.tsx            # Главная (ru/en/hy)
│   ├── about/page.tsx
│   ├── services/page.tsx
│   ├── services/[slug]/page.tsx
│   ├── team/page.tsx
│   ├── team/[slug]/page.tsx
│   ├── blog/page.tsx
│   ├── blog/[slug]/page.tsx
│   ├── cases/page.tsx
│   ├── cases/[slug]/page.tsx
│   └── contact/page.tsx
├── admin/                  # Остаётся без [locale]
└── api/                    # Остаётся без [locale]
```

### 6.3 Изменения в data.ts
Передавать `locale` в каждую функцию вместо константы `LANG = 'ru'`:
```ts
export async function getHeroSlides(lang = 'ru') { ... }
export async function getServices(lang = 'ru') { ... }
// и т.д.
```

### 6.4 Переключатель языков в Header
- Текущий Header показывает языки декоративно (не меняет URL)
- Нужно: `useRouter().push('/' + locale + pathname)` при выборе языка
- Текущий язык определять из URL (`useParams().locale`)

### 6.5 Middleware
```ts
// src/middleware.ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';
export default createMiddleware(routing);
export const config = { matcher: ['/((?!api|admin|_next|.*\\..*).*)'] };
```

**Редиректы:**
- `/` → `/ru/` (язык по умолчанию)
- `/services` → `/ru/services`
- `/admin/*` — без изменений

## ФАЗА 9 — SEO + Финальная полировка

### 9.1 Мета-теги (SEO)

Добавить `generateMetadata()` в каждую страницу:

```ts
// src/app/[locale]/page.tsx
export async function generateMetadata() {
  const settings = await getSettings();
  return {
    title: settings.site_name,
    description: settings.site_description,
    openGraph: { title: settings.site_name, ... },
  };
}

// src/app/[locale]/services/[slug]/page.tsx
export async function generateMetadata({ params }) {
  const service = await getServiceBySlug(params.slug);
  return { title: `${service.title} | LSA Legal`, description: service.intro[0] };
}
```

### 9.2 Sitemap
- Создать `src/app/sitemap.ts` — автогенерация из БД
- Все услуги, члены команды, посты блога, дела

### 9.3 robots.txt
- Создать `src/app/robots.ts`
- Закрыть `/admin/*` от индексации

### 9.4 Dashboard в админке
Текущий `/admin` — пустой. Добавить карточки:
- Количество услуг / постов / дел / членов команды
- Последние добавленные записи
- Быстрые ссылки

### 9.5 Оптимизация изображений
- Добавить `sizes` prop к компонентам `<Image>`
- Добавить `placeholder="blur"` для карточек команды
- Проверить `priority` для hero изображений

### 9.6 Обработка ошибок
- Создать `src/app/[locale]/not-found.tsx` — страница 404
- Создать `src/app/[locale]/error.tsx` — страница ошибки
- Проверить все `notFound()` вызовы

### 9.7 Кеширование
- Добавить `revalidate` к Server Components для контента который редко меняется:
```ts
export const revalidate = 3600; // 1 час
```
- Или ISR с `revalidatePath('/services')` из API роутов при сохранении
